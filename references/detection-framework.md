# Pareto Startup Thinking Framework

## Goal

Find the smallest set of actions, bets, or decisions most likely to move a startup outcome under real constraints.

The point is not to sound focused. The point is to improve the actual bottleneck.

## Ranking Order

Use this order:

1. Bottleneck relevance
2. Expected impact
3. Cost of delay
4. Reversibility and learning speed
5. Carry-cost added

## Required Questions

Before prioritizing, define:

- the outcome
- the time horizon
- the current bottleneck
- what `good enough` means right now

Without those, prioritization turns into random trimming.

## Leverage Rubric

For each candidate, judge:

- `impact`
- `confidence`
- `effort`
- `cost-of-delay`
- `carry-cost`
- `bottleneck-fit`

You do not need fake numeric precision. Clear relative judgment is enough.

## False-Focus Checks

Reject or down-rank work when:

- it looks important but does not touch the bottleneck
- it improves a local metric while missing the real outcome
- it adds complexity that future work will have to carry
- it is easy to talk about but not load-bearing
- it is being prioritized mainly because it is visible or already in motion

## Action Types

For each high-leverage item, choose one:

- `do now`
- `test first`
- `measure first`
- `defer`
- `delete`
- `simplify`

## PM-Friendly Default

A good answer usually says:

- what matters most
- why it matters most
- what to stop doing
- what the next smallest move is
